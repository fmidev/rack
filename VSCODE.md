# VSCode development setup for Rack

## Prerequisites

```bash
apt install git make g++ bear
apt install libhdf5-dev libpng-dev libproj-dev libtiff-dev libgeotiff-dev libgomp1
```

Install the **C/C++ extension** (`ms-vscode.cpptools`) in VSCode.

## First-time setup

```bash
cd src
make configure   # detects libraries, writes install-rack.cnf
```

## Build

| Method | Command |
|--------|---------|
| Shell | `make` or `make -j$(nproc)` (from repo root or `src/`) |
| VSCode | **Ctrl+Shift+B** → *Build Rack* |

## Tasks (VSCode Terminal → Run Task)

| Task | Description |
|------|-------------|
| Build Rack | Incremental release build (`-O3`) |
| Build Rack (Debug) | Debug build (`-O0 -g3`), required before F5 |
| Clean | Remove `src/Build/` and executable |
| Rebuild | Clean + Build |
| Configure | Re-run `configure.sh` |
| Generate compile_commands.json | Run `bear -- make` for precise IntelliSense |

## Debugging

Press **F5** — pre-builds in debug mode, then launches gdb inside VSCode.
Use *Debug Rack (no rebuild)* for fast iteration when the binary is already up to date.

## IntelliSense

Two configurations are selectable in the VSCode status bar:

- **Linux (system libraries)** — uses `/usr/include/hdf5/serial`, `/usr/include/geotiff`, etc.
- **Linux (conda env)** — uses `$HOME/.conda/envs/rack-install/include`

Both configurations automatically use `src/compile_commands.json` when it exists,
which gives file-accurate IntelliSense. Generate it once with:

```bash
make compdb   # requires: sudo apt install bear
```

## File layout

```
rack/
├── Makefile                  root convenience wrapper (delegates to src/)
├── src/
│   ├── Makefile              main build rules (reads install-rack.cnf)
│   ├── install-rack.cnf      generated by make configure — not in git
│   ├── Build/                compiled objects — not in git
│   └── rack                  built executable — not in git
└── .vscode/
    ├── tasks.json            build / clean / configure tasks
    ├── c_cpp_properties.json IntelliSense include paths
    ├── launch.json           gdb debug configurations
    └── settings.json         file exclusions, associations
```

## Makefile reference

```
make                  build (default)
make -j$(nproc)       parallel build
make configure        (re-)generate install-rack.cnf
make clean            remove Build/ and rack executable
make install          install to prefix in install-rack.cnf
make compdb           generate compile_commands.json (needs bear)
make cppcheck         static analysis
make docker-image     build Docker image
```

## Notes

- `install-rack.cnf` supports two formats: direct `CCFLAGS`/`LDFLAGS` variables
  (output of `configure.sh`) or individual path variables `HDF5_INCLUDE`, `HDF5_LIB`, …
  (output of `configure-venv.sh`). The Makefile handles both.
- The original `build.sh` / `make.sh` scripts are untouched and remain usable
  without Make or any IDE.
- Dependency tracking uses g++ `-MMD -MP` flags: only files affected by a changed
  header are recompiled. No IDE is required for this to work.
